## 1. フィードバック収集機能（2段階レビューフロー）

### 仕様
- 学習記録が2日連続になった時にモーダル表示
- 一度表示したら再表示しない（AsyncStorageでフラグ管理）

### フロー
1. モーダル表示「DORILOOPの使い心地はどうですか？」
2. 3つのボタン: [😊 いい感じ] [😐 微妙] [😞 使いにくい]
3. 分岐:
   - 😊 いい感じ → expo-store-review でApp Storeレビューを依頼
   - 😐 微妙 / 😞 使いにくい → テキスト入力画面「改善点を教えてください」→ 送信ボタン（送信先は後で設定、今はconsole.logでOK）
4. 完了後「ありがとうございます！」と表示してモーダルを閉じる

### 必要なパッケージ
- expo-store-review

## 2. 広告表示（無料版のみ）

### 仕様
- Google AdMobを使用
- バナー広告を画面下部に表示
- プレミアム会員（usePurchasesのisPremiumがtrue）の場合は非表示

### 対象画面
- ホーム画面
- 問題一覧画面
- 分析画面

### 必要なパッケージ
- react-native-google-mobile-ads

### 広告ユニットID
- テスト用IDを使用（本番IDは後で差し替え）
- iOS: ca-app-pub-3940256099942544/2934735716（テスト用バナー）

### 実装の注意
- isPremiumがtrueの場合は広告コンポーネントを表示しない
- 広告の読み込み中・エラー時は何も表示しない

## 3. 最終ログイン日の記録

### 仕様
- Supabaseのusersテーブル（またはprofilesテーブル）に `last_active_at` カラムを追加
- アプリ起動時またはログイン時に現在日時を更新
- 継続率計測に使用

### 実装
- アプリのルートコンポーネント（_layout.tsxなど）で、認証済みユーザーのlast_active_atを更新
